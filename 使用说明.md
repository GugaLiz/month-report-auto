# 月报填写工具 - 使用说明

## 两种运行方式

由于您的 Node 版本较低（14.14.0），项目提供了两种运行方式：

### 方式一：独立 HTML 版本（推荐，无需构建）

直接在浏览器中打开文件即可使用：

1. **直接打开文件**
   ```
   双击打开 standalone.html 文件
   ```

2. **或使用简单的 HTTP 服务器**（推荐，避免跨域问题）
   ```bash
   # 使用 Python 3
   python -m http.server 3000
   
   # 或者使用 Python 2
   python -m SimpleHTTPServer 3000
   
   # 然后在浏览器中访问
   http://localhost:3000/standalone.html
   ```

### 方式二：Vue + Vite 构建版本（需要 Node 18+）

如果您升级 Node 到 18+ 或 20+，可以使用完整的开发环境：

1. **安装依赖**
   ```bash
   npm install
   ```

2. **启动开发服务器**
   ```bash
   npm run dev
   ```

3. **构建生产版本**
   ```bash
   npm run build
   ```

## 升级 Node 版本

**推荐升级 Node 版本以获得最佳开发体验。**

### Windows 系统升级方法：

1. **访问 Node.js 官网**
   - 下载地址：https://nodejs.org/
   - 推荐下载 LTS 版本（当前为 v20.x）

2. **或使用 nvm-windows**（推荐）
   ```powershell
   # 安装 nvm-windows
   # 下载地址：https://github.com/coreybutler/nvm-windows/releases
   
   # 安装 Node 20
   nvm install 20
   
   # 使用 Node 20
   nvm use 20
   
   # 验证版本
   node -v
   ```

## 功能说明

### 1. 配置 Cookie

首次使用需要配置 Worktile Cookie：

1. 打开浏览器，访问 Worktile 并登录
2. 按 F12 打开开发者工具
3. 切换到 Network（网络）面板
4. 刷新页面
5. 点击任意请求
6. 找到 Request Headers（请求头）
7. 复制 Cookie 的完整内容
8. 在工具的"设置"中粘贴

### 2. 使用流程

1. **选择日期范围**
   - 手动选择开始和结束日期
   - 或点击"本月"按钮快速选择当月

2. **获取数据**
   - 点击"获取工时数据"按钮
   - 系统自动调用 Worktile API

3. **查看结果**
   - 重点工作清单：自动提取本月完成的需求
   - K线用时统计：自动分类并计算工作天数
   - 工作自评：生成3个参考版本
   - 本月工作日：自动计算（已排除周末和节假日）

4. **复制导出**
   - 点击"复制表格"或"复制统计"
   - 内容自动复制到剪贴板
   - 可直接粘贴到文档中

### 3. K线分类规则

工具会根据需求标题自动分类到对应 K线：

| K线 | 关键字 |
|-----|--------|
| K1 | 剑网3 |
| K2 | 剑网一、剑网2、剑侠情缘、剑侠世界 |
| K3 | 尘白 |
| K8 | 解限机 |
| SSG | SEED、星砂岛 |

### 4. 数据存储

所有设置和 Cookie 都存储在浏览器的 LocalStorage 中：
- 不会上传到服务器
- 仅在本地使用
- 可随时清除

## 常见问题

### Q1: 提示 "Cookie 无效" 怎么办？
A: Cookie 会过期，需要重新从浏览器获取并更新。

### Q2: 跨域请求被阻止？
A: 建议使用 HTTP 服务器运行（如上述 Python 命令），而不是直接打开 HTML 文件。

### Q3: 数据不准确？
A: 检查配置的项目 ID 和成员 ID 是否正确。

### Q4: 工作日计算不对？
A: 节假日数据需要每年更新，当前包含 2026 年的节假日信息。

## 技术支持

如有问题，请检查：
1. Cookie 是否有效
2. 项目 ID 和成员 ID 是否正确
3. 浏览器控制台是否有错误信息
4. 网络连接是否正常
