# 月报填写工具 - 需求文档

## 一、项目背景

作为前端开发人员，每个月需要填写月报，包括：
- 本月重点工作清单
- 个人工作自评
- K线工作用时统计

目前月报填写过程中，需要手动从 Worktile 平台查看工时记录，手动整理和计算，效率较低。因此需要开发一个自动化工具，从 Worktile API 获取数据并自动生成月报内容。

## 二、核心功能需求

### 2.1 工时数据获取

#### 2.1.1 接口信息
- **接口地址**: `https://webxsj.worktile.com/api/mission-vnext/workload/report/workload-day-detail/group`
- **请求方法**: POST
- **参数说明**:
  ```json
  {
    "created_at": {
      "start": 1770566400,  // 开始时间戳（秒）
      "end": 1770998399      // 结束时间戳（秒）
    },
    "reported_at": {
      "start": null,
      "end": null
    },
    "project_ids": [
      "66a9fd0526a6ce0007bc2c73",
      "621ed84824cdc85eafdf9195"
    ],
    "member_ids": ["0fa3263e47624b1e8b40e278a93bd074"],
    "group_by": "day",
    "entry_type": 1,
    "type_ids": []
  }
  ```

#### 2.1.2 用户输入
- 开始日期（YYYY-MM-DD 格式）
- 结束日期（YYYY-MM-DD 格式）
- 自动转换为时间戳发送请求

#### 2.1.3 请求配置
需要在请求头中携带以下信息：
```
headers: {
  "accept": "application/json, text/plain, */*",
  "content-type": "application/json",
  "cookie": "用户登录凭证"
}
```

### 2.2 重点工作清单生成

#### 2.2.1 数据提取
从接口响应中提取需求名称：
- **数据路径**: `data.references.tasks[].parent.title`
- **示例数据**: "《剑网一》端游2026年春节官网换肤需求"

#### 2.2.2 输出格式
生成 Markdown 表格：

| 时间 | 主要工作事项 | 负责人 | 完成进度 | 个人自评 |
|------|-------------|--------|---------|---------|
| 2月 | 《剑侠世界4》官方、通道PC站建站 | | 100% | |
| | 《剑侠世界4》国内集成小程序 | | 100% | |
| | 《剑网3》26年2月网站换肤 | | 100% | |

### 2.3 工作自评生成

#### 2.3.1 功能描述
基于本月需求列表，使用 AI 生成 3 个工作自评参考文本，方便填写。

#### 2.3.2 自评内容要求
- 字数：100-150字
- 内容方向：
  - 突出完成的工作量和项目数量
  - 体现个人技术能力提升
  - 强调对业务需求的支持和配合
  - 提及遇到的困难及解决方案（可选）

#### 2.3.3 参考示例
```
本月主要完成网页制作年前需求上线。要梳理在做和正在上线的业务工作，任凭和业务方沟通时能
更多掌握节奏，任凭业务工作的同时积极协助团队其他成员业务上线，并逐步落地相关内容。
```

### 2.4 K线用时统计

#### 2.4.1 K线分类规则

| K线类型 | 关键字匹配规则 |
|---------|---------------|
| K1 | 剑网3、剑网三 |
| K2 | 剑网一、剑网1、剑网2、剑网二、剑侠情缘、剑侠世界 |
| K3 | 尘白 |
| K8 | 解限机、解限 |
| SSG | SEED、seed、星砂岛 |

#### 2.4.2 计算逻辑

1. **关键字匹配**
   - 从 `data.references.tasks[].parent.title` 提取需求标题
   - 根据关键字判断属于哪个 K 线
   - 如果同一需求包含多个关键字，按优先级匹配（K1 > K2 > K3 > K8 > SSG）

2. **工时统计**
   - 从 `data.references.entries[]` 获取每条工时记录
   - 字段：`duration`（单位：小时）
   - 通过 `task_id` 关联到具体需求

3. **转换天数**
   - 每天按 8 小时计算
   - 公式：`天数 = Math.round(总小时数 / 8 * 10) / 10`（保留一位小数）

#### 2.4.3 输出格式
```
K1: 2天
K2: 5天
K3: 0.5天
K8: 1.5天
年假: 1天
调休: 0天
总计: 16天
```

### 2.5 工作日天数计算

#### 2.5.1 计算规则
- 当月总天数
- 减去周末（周六、周日）
- 减去法定节假日（根据中国万年历）

#### 2.5.2 节假日数据源
- 2025年节假日安排（国务院公布）
- 2026年节假日安排（需要动态更新）

#### 2.5.3 常见节假日
- 元旦：1月1日
- 春节：农历正月初一至初三（含调休）
- 清明节：清明当天
- 劳动节：5月1日
- 端午节：农历五月初五
- 中秋节：农历八月十五
- 国庆节：10月1日至7日

### 2.6 需求调整与优化（2026-03-05 测试反馈）

#### 2.6.1 重点工作清单表格简化

**问题描述**：
当前重点工作清单表格包含 5 列（时间、主要工作事项、负责人、完成进度、个人自评），字段较多，实际填写时只有"主要工作事项"是必需的。

**调整方案**：
- 表格只保留"主要工作事项"一列
- 简化输出格式，便于快速查看和复制
- 移除不必要的空列（负责人、完成进度、个人自评）

**调整后的输出格式**：
保持简单表格：

| 主要工作事项 |
|-------------|
| 《剑侠世界4》官方、通道PC站建站 |
| 《剑侠世界4》国内集成小程序 |
| 《剑网3》26年2月网站换肤 |

#### 2.6.2 K线用时统计数据问题

**问题描述**：
点击"获取工时数据"后，K线用时统计模块没有显示数据。

**排查方向**：
1. 检查 API 返回的数据结构是否正确
2. 验证任务标题提取逻辑（`task.parent.title`）
3. 确认 K线分类算法是否正常工作
4. 检查工时汇总计算是否有bug

**可能原因**：
- 数据解析路径错误
- 任务没有 parent 属性或 parent.title 为空
- K线关键字匹配失败
- 工时数据累加逻辑问题

**解决方案**：
- 添加详细的日志输出，追踪数据处理流程
- 增加容错处理，当 parent 不存在时使用 task.title
- 优化 K线分类逻辑，支持多种数据格式

#### 3.2.3 数据解析模块
```javascript
function parseWorkloadData(data) {
  // 1. 提取需求列表
  const tasks = data.references.tasks;
  
  // 2. 提取工时记录
  const entries = data.references.entries;
  
  // 3. 关联任务和工时
  const taskTimeMap = entries.reduce((map, entry) => {
    const taskId = entry.task_id;
    const duration = entry.duration;
    map[taskId] = (map[taskId] || 0) + duration;
    return map;
  }, {});
  
  return { tasks, taskTimeMap };
}
```

#### 3.2.4 K线分类模块
```javascript
const K_LINE_RULES = {
  K1: ['剑网3'],
  K2: ['剑网一', '剑网2', '剑侠情缘', '剑侠世界'],
  K3: ['尘白'],
  K8: ['解限机'],
  SSG: ['SEED', '星砂岛']
};

function classifyTask(title) {
  for (const [kLine, keywords] of Object.entries(K_LINE_RULES)) {
    if (keywords.some(keyword => title.includes(keyword))) {
      return kLine;
    }
  }
  return 'OTHER';
}

function calculateKLineHours(tasks, taskTimeMap) {
  const kLineHours = {};
  
  tasks.forEach(task => {
    const title = task.parent?.title || '';
    const kLine = classifyTask(title);
    const hours = taskTimeMap[task._id] || 0;
    
    kLineHours[kLine] = (kLineHours[kLine] || 0) + hours;
  });
  
  // 转换为天数
  const kLineDays = {};
  for (const [kLine, hours] of Object.entries(kLineHours)) {
    kLineDays[kLine] = Math.round(hours / 8 * 10) / 10;
  }
  
  return kLineDays;
}
```

#### 3.2.5 工作日计算模块
```javascript
function calculateWorkDays(year, month) {
  const firstDay = new Date(year, month - 1, 1);
  const lastDay = new Date(year, month, 0);
  const totalDays = lastDay.getDate();
  
  let workDays = 0;
  
  for (let day = 1; day <= totalDays; day++) {
    const date = new Date(year, month - 1, day);
    const dayOfWeek = date.getDay();
    
    // 跳过周末
    if (dayOfWeek === 0 || dayOfWeek === 6) {
      continue;
    }
    
    // 跳过法定节假日
    if (isHoliday(date)) {
      continue;
    }
    
    workDays++;
  }
  
  return workDays;
}

function isHoliday(date) {
  // 从配置或 API 获取节假日数据
  const holidays = getHolidayList(date.getFullYear());
  const dateStr = formatDate(date, 'YYYY-MM-DD');
  return holidays.includes(dateStr);
}
```

## 八、风险评估

### 8.1 技术风险
- **Cookie 失效**: Worktile Cookie 可能过期，需要用户重新登录
- **API 变更**: Worktile API 可能变更，需要及时适配
- **跨域问题**: 浏览器可能阻止跨域请求

**应对措施**:
- 提供 Cookie 更新入口
- 监控 API 变化，及时更新
- 考虑使用浏览器插件或代理服务器

### 8.2 数据风险
- **数据不准确**: K线分类可能存在误判
- **节假日数据**: 节假日数据需要及时更新

**应对措施**:
- 支持手动修正 K线分类
- 定期更新节假日配置文件
